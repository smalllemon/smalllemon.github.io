Node的编码规范

1、遵守规则

无论是何种编程语言，首先最重要的是约束自己的代码规范，使之能够在团队中易维护并避免低级错误。

2、编码规范

(1)、空格与格式

    a.缩进

      采用2个空格进行缩进，而不是tab缩进。

    b.变量声明

      永远使用var声明每一个变量，在node中不允许省略var关键字

      如：var  assert = require("assert");

          var  fork = require("child_process").fork;

          var  net  = require("events").EventEmitter;


          错误的实例：

           var  assert  = require("assert")
           , fork = require("child_process").fork
           , net  = require("events").EventEmitter;

    c.空格

      在所有常见的操作符前后都需要添加一个空格

      var  foo = "bar" + baz;


      if (true) {
         //some code
      }


(2)、在node中尽可能使用单引号

(3)、大括号的位置

    if (true) {
         //some code
    }

(4)、逗号的位置

    逗号用于变量声明的分隔或者是元素的分隔。如果逗号不在行结尾，后面需要一个空格。

    切忌逗号不允许出现在行首的位置处。

    var  obj = { foo:"hello", boo:"world" };

(5)、分号

     分号一般情况下就表示该语句到此结束，在必要的是时候就需要为它添加上分号。

3、命名规范

在编码过程中，命名是重头戏，好的命名可以令代码赏心悦目，带来愉悦的阅读享受，令代码具有良好的可维护性。

a. 变量命名

  在node中变量名一律都采用小驼峰式命名，即：除了第一个单词的首字母不大写外，每个单词的首字母都大写。

  var  addminUser = {} ;

b. 方法命名

  方法命名和变量命名遵循同一种规则，尽量采用动词或判断性词汇。

  如：var getUser = function () {} ;


c. 类命名

   一般情况下类名采用大驼峰式命名

   function User {

   }

    在比较操作中，请使用 === 代替 ==
