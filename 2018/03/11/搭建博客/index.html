Coding every day

http://lowrank.science/Hexo-Github/

2、前提条件

安装hexo之前，必须确认本地已经安装了Node.js和Git。

3、 本地部署(命令操作步骤)

本地磁盘新建文件

npm install -g hexo-cli

npm config set registry "https://registry.npm.taobao.org"

hexo init

npm  install

hexo generate ( 生成静态资源)

hexo server ( http://localhost:4000/ 本地预览)

3、 git配置

(1)、github仓库中新建一个仓库，用于线上存放博客资源

(2)、git config --global user.name "YOUR NAME"

(3)、git config --global user.email "YOUR EMAIL ADDRESS"

(4)、ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

(5)、eval "$(ssh-agent -s)"

(6)、ssh-add ~/.ssh/id_rsa

(7)、clip < ~/.ssh/id_rsa.pub

(8)、仓库创建秘钥

(9)、ssh -T git@github.com (测试连接，成功之后提示用户信息)

4、 部署流程

(1)、Hexo配置

    用编辑器打开博客目录下的配置文件_config.yml,找到如下属性并进行修改：

    deploy:
      type: git
      repo: git@github.com:smalllemon/smalllemon.github.io.git
      branch: master

(2)、完成部署

    $ npm install hexo-deployer-git --save

    $ hexo generate

    $ hexo deploy

自此，博客工作完成。

5、日常操作

(1)、写文章

    $ hexo new [layout] "文章标题" #新建文章

    执行完上述new命令后，生成指定名称的文章至 ..\source_posts\文章标题.md,然后用编辑器打开“文章标题.md”按照Markdown语法书写文章.


(2)、提交

    每次在本地对博客进行修改后，先执行下列命令提交网站相关的文件。

    $ git add .

    $ git commit -m "..."

    $ git push origin develop

    然后才执行hexo generate -d发布网站到master分支上。

     $ hexo g

     $ hexo d

6、hexo命令提示tips

hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help  # 查看帮助
hexo version  #查看Hexo的版本
hexo deploy -g  #生成加部署
hexo server -g  #生成加预览
#命令的简写
hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy
