Node的安装和调试

1、Node的安装

进入官网选择合适的版本，下载安装包，根据提示进行安装

安装完成之后，打开系统的命令行

$ node -v 显示出当前node的版本号

2、调试node

在进行调试前，需要通过debugger语句在代码中设置断点，这样在执行时就会形成中断。

3、项目工程化

在项目工程化中最重要的核心是：目录结构 构建工具 编码规范 代码审查。

(1)、部署流程

代码在完成开发、审查、合并之后才会进入到部署流程：部署环境  部署操作

(2)、动静分离

通常情况下，会将图片、脚本、样式表和多媒体等静态文件放置在专业的静态文件服务器上，
让node只负责动态请求即可。
将项目中所涉及的静态资源单独使用静态的额服务器进行存储。动静分离可以提高性能。

(3)、日志

在项目中通常会出现各种各样的问题，此时就需要使用到日志，可以快速定位到问题，以便更快的解决问题。

(4)、访问日志

用来记录每个客户端对应用的访问。在node开发的web应用中，可以自行实现访问日志的记录。

var  app = connect();

connect.logger.format("相关的参数");

app.use(connect.logger({
    format:"home",
    stream: fs.createWriteStream(_dirname + "/logs/access.log")
}));
